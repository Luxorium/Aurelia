# Aurelia 0.2.2 - Zero-Argument Launch and server.properties

## Highlights

- **`./aurelia-server` just works.** No flags required. On first run, Aurelia generates a commented `server.properties`, creates a `./world` flat world, binds to `0.0.0.0:25565`, and is ready for a Beta 1.7.3 client.
- **`server.properties` support.** Standard vanilla-style configuration file read from the working directory. Supported keys: `server-port`, `server-ip`, `level-name`, `motd`, `max-players`, `online-mode`, `view-distance`. Keys `spawn-protection`, `white-list`, `allow-nether`, `difficulty`, and `gamemode` are parsed with a warning but not yet enforced. Unknown keys are silently ignored and never crash the server.
- **Precedence:** CLI flag → `server.properties` → built-in defaults.
- **Existing vanilla Beta 1.7.3 worlds:** place your world folder at `./world` (must contain `level.dat` and `region/*.mcr`), then run `./aurelia-server`. The vanilla format is auto-detected.
- 47 new tests; 190 total across the workspace, all passing.
- Pre-existing clippy warnings fixed across `aurelia-protocol` and `aurelia-world`.

## Quick Start

```bash
# Build
cargo build --release
cp target/release/aurelia-server .

# Run (generates server.properties and ./world on first run)
./aurelia-server
```

## server.properties Reference

| Key | Default | Notes |
|-----|---------|-------|
| `server-port` | `25565` | TCP listen port |
| `server-ip` | *(blank)* | Bind address; blank = `0.0.0.0` |
| `level-name` | `world` | World folder path |
| `motd` | `A Minecraft Server` | Server list message |
| `max-players` | `20` | Parsed; not yet enforced |
| `online-mode` | `false` | Must be `false`; session auth not implemented |
| `view-distance` | `1` | Chunk radius sent to clients (0–8) |

Keys parsed but not yet enforced (warned at startup): `spawn-protection`, `white-list`, `allow-nether`, `difficulty`, `gamemode`.

## Known Missing Systems

- Verified production login response semantics.
- Crafting, chests, furnaces, workbench UI, full inventory rules.
- Item entities, pickups, mobs, AI, visible entity packets, combat.
- Vanilla terrain generation, worldgen features, fluids, redstone, weather, sleep, permissions, vanilla commands.
- Full vanilla McRegion/NBT world and player save parity.
- `max-players`, `spawn-protection`, `white-list`, `difficulty`, `gamemode` enforcement.

## Downloads

A prebuilt Linux x86_64 `aurelia-server` binary is attached to the GitHub release. On other platforms, build from source:

```bash
cargo build --release -p aurelia-server
```

## Compatibility Notes

This release is a compatibility foundation, not a complete Beta 1.7.3 server. The real-client join path is stable for basic testing (movement, break/place, chat, persistence) but authentication, terrain generation, containers, and entity parity are not implemented.
